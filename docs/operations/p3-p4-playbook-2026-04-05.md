# P3-P4 Playbook (Indexes + Smoke)

## Scope
- Point 3: apply missing DB index migration in target environment.
- Point 4: run final smoke using the release checklist.

## 1) Migration To Apply (Point 3)

Migration file:
- `apps/api/src/db/migrations/versions/1183644aa765_add_analysis_results_composite_indexes.py`

Indexes created:
- `ix_analysis_results_team_id_created_at`
- `ix_analysis_results_team_id_user_id_created_at`
- `ix_analysis_results_team_id_code_hash_created_at`

## 2) Apply Migration In Target Environment

From `apps/api` with the correct target `DATABASE_URL`:

```bash
cd apps/api
source .venv/bin/activate
alembic current
alembic heads
alembic upgrade head
alembic current
```

If you use Railway, run the same commands in the environment/session where `DATABASE_URL` points to Railway Postgres.

## 3) Verify Indexes Exist

Run this SQL in Postgres:

```sql
SELECT indexname, indexdef
FROM pg_indexes
WHERE tablename = 'analysis_results'
  AND indexname IN (
    'ix_analysis_results_team_id_created_at',
    'ix_analysis_results_team_id_user_id_created_at',
    'ix_analysis_results_team_id_code_hash_created_at'
  )
ORDER BY indexname;
```

Expected: 3 rows.

## 4) Quick Smoke Run (Point 4)

Use the full checklist in:
- `docs/releases/web-mvp-checkpoint-2026-04-05.md`

Recommended quick subset (10-15 min):

```bash
# API targeted tests
cd apps/api
source .venv/bin/activate
pytest tests/test_analytics.py tests/test_teams.py -q

# Web targeted tests
cd ../web
npx vitest run src/test/team-insights-v3.test.tsx src/test/team-analytics.test.tsx
```

Manual checks:
- `/health` returns OK.
- Team analytics summary/insights happy path returns 200.
- Non-member access returns 403.
- Invalid `days` / `top_n` as member returns 422.
- Selector clicks emit `team_insights_selector_changed` events.
- Error state shows retry and recovers after API comes back.

## 5) Blocker Notes

If `alembic upgrade head` fails with auth errors locally, do not treat migration as failed.
It only means your local DB credentials are wrong for that shell.
Run the same command in the actual destination environment with valid `DATABASE_URL`.
