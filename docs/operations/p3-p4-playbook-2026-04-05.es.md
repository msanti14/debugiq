# Playbook P3-P4 (Indices + Smoke)

## Alcance
- Punto 3: aplicar la migracion faltante de indices en el entorno destino.
- Punto 4: ejecutar el smoke final usando el checklist de release.

## 1) Migracion a aplicar (Punto 3)

Archivo de migracion:
- `apps/api/src/db/migrations/versions/1183644aa765_add_analysis_results_composite_indexes.py`

Indices creados:
- `ix_analysis_results_team_id_created_at`
- `ix_analysis_results_team_id_user_id_created_at`
- `ix_analysis_results_team_id_code_hash_created_at`

## 2) Aplicar migracion en el entorno destino

Desde `apps/api` con `DATABASE_URL` correcto para el entorno objetivo:

```bash
cd apps/api
source .venv/bin/activate
alembic current
alembic heads
alembic upgrade head
alembic current
```

Si usas Railway, ejecuta esos mismos comandos en la sesion/entorno donde `DATABASE_URL` apunte al Postgres de Railway.

## 3) Verificar que los indices existen

Ejecuta este SQL en Postgres:

```sql
SELECT indexname, indexdef
FROM pg_indexes
WHERE tablename = 'analysis_results'
  AND indexname IN (
    'ix_analysis_results_team_id_created_at',
    'ix_analysis_results_team_id_user_id_created_at',
    'ix_analysis_results_team_id_code_hash_created_at'
  )
ORDER BY indexname;
```

Esperado: 3 filas.

## 4) Smoke rapido (Punto 4)

Usar checklist completo en:
- `docs/releases/web-mvp-checkpoint-2026-04-05.md`

Subconjunto recomendado (10-15 min):

```bash
# API targeted tests
cd apps/api
source .venv/bin/activate
pytest tests/test_analytics.py tests/test_teams.py -q

# Web targeted tests
cd ../web
npx vitest run src/test/team-insights-v3.test.tsx src/test/team-analytics.test.tsx
```

Checks manuales:
- `/health` devuelve OK.
- Team analytics summary/insights happy path devuelve 200.
- Acceso de no miembro devuelve 403.
- `days` / `top_n` invalidos como miembro devuelven 422.
- Clicks en selectores emiten eventos `team_insights_selector_changed`.
- Estado de error muestra retry y recupera al volver API.

## 5) Nota de bloqueo comun

Si `alembic upgrade head` falla por autenticacion local, no significa que la migracion este mal.
Solo indica que las credenciales locales no son las correctas para ese shell.
Ejecuta el mismo comando en el entorno destino con `DATABASE_URL` valido.
